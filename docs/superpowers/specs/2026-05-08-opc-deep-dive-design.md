## 一、背景与目标

### 1.1 背景

2025-2026 年，AI 驱动的"一人公司"（One Person Company, OPC）在中国和全球范围内爆发式增长。新华社、中国日报、中国经营报等主流媒体密集报道，上海临港等地方政府出台专项扶持政策。AI 编程工具（Claude Code / Cursor / Copilot 等）的成熟使单人开发全栈产品成为可能，AI Agent 让一个人可以运营"AI 智能体集群"替代传统团队。

然而，市场上缺乏系统化的学习资料——大多数内容停留在"OPC 是什么"的趋势层面，或散落在个人博客和播客中。本深度指南的目标是填补这一空白。

### 1.2 目标

在 `config/deep-dive/` 下新建 `opc/` 目录，创建包含 index.md + 14 篇深度文章的完整深度指南，与现有四个深度指南（agent-skills、context-management、openclaw、deepseek-v4）同级并列。

### 1.3 目标读者

- 正在或打算成为 OPC 的独立开发者 / 程序员
- 希望借助 AI 工具开启一人创业的技术从业者
- 对"超级个体"商业模式感兴趣的各行业从业者
- 已完成基础模块学习，希望将 AI 能力转化为商业产出的学习者

### 1.4 核心框架

以**双杠杆公式（代码 × 媒体）**为主线，辅以认知底座和生存系统，构建完整的 OPC 知识体系：

```
代码杠杆（怎么做产品）× 媒体杠杆（怎么让人知道）
        ↓                          ↓
   技术实操                  内容+增长+品牌
        ↓                          ↓
   MVP → 架构 → 自动化 → 数据    BIP → 获客 → 变现

        ↑        认知底座（为什么做、做什么）        ↑
        ↑        生存系统（商业模式、法务、心理）      ↑
```

## 三、写作方式

### 3.1 写作方法

- **搜索驱动**: 使用 unified-search 技能进行网络搜索，确保内容基于 2025-2026 年的最新信息
- **视角指导**: 使用 huashu-perspective 花叔视角指导写作风格——短句、结论前置、数据锚定、案例驱动

### 3.2 每篇文章的组成要素

- **开篇摘要**: 先给出结论，再展开论述
- **分层组织**: 每个主题使用表格/列表结构化呈现
- **案例植入**: 每个核心观点附带具体案例（OPC 真实案例、数据支撑）
- **行动建议**: 每篇末尾包含读者可以立刻操作的行动项
- **检验标准**: 对标现有 deep-dive 模块，每篇末尾带自测问题

### 3.3 字数与深度

- 每篇约 25-50KB（对标现有模块规格）
- 深度与广度平衡：既有框架性知识，也有可操作的步骤

1. `config/deep-dive/index.md` → 添加 OPC 深度指南条目（含简介、适合读者、14 篇概览）
