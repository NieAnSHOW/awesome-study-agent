# 技术研发案例

> **学习目标**: 掌握 WorkBuddy 在技术研发场景中的实际应用
>
> **预计时间**: 30 分钟
>
> **难度等级**: ⭐⭐⭐☆☆

本章收录 3 个技术研发场景案例，涵盖代码审查、技术文档生成和日志自动化处理等任务。

---

## 案例 1：代码审查

**一句话**: 让 WorkBuddy 审查本地代码，发现潜在问题。

**适用人群**: 开发者

**前置条件**: 本地有待审查的代码目录

**指令示例**:

```
审查 /Users/me/project/src 目录下的所有 Python 代码，检查潜在的 bug、安全漏洞、性能问题和代码风格违规，输出审查报告到桌面。
```

**交付成果**: 代码审查报告（含问题列表、严重程度、修改建议）

---

## 案例 2：技术文档生成

**一句话**: 让 WorkBuddy 基于代码自动生成 API 接口文档。

**适用人群**: 开发者、技术写作

**前置条件**: 有完整的项目代码

**指令示例**:

```
阅读 /Users/me/project 的代码，分析其中的 API 接口和主要函数，生成一份 API 接口文档，包含接口说明、参数列表、返回值示例。
```

**交付成果**: Markdown 格式的 API 文档

---

## 案例 3：日志自动化处理

**一句话**: 让 WorkBuddy 自动归档和分析日志文件。

**适用人群**: 运维、开发者

**前置条件**: 有需要处理的日志目录

**指令示例**:

```
把 /logs 目录下的所有 .log 文件按日期（年/月）创建文件夹归档，然后统计每个日志文件中的 ERROR 级别错误数量，输出汇总报告。
```

**交付成果**: 归档后的目录结构 + 错误统计报告

---

## 扩展思路

更多技术场景：
