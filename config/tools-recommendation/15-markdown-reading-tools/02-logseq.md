# Logseq

## Logseq 是什么

Logseq 是一款开源的大纲式知识管理工具。不是传统的文档编辑器——它的每一行都是一个独立的"块"（Block），可以单独引用、单独搬运。

一句话说清楚：**把你的笔记拆成一块一块来写，每一块都能精确引用。**

你写的所有内容都是纯文本 Markdown 文件，存在你自己电脑上。加上 MIT 开源协议，Logseq 完全免费——没有付费墙，没有隐藏功能。

## 核心特点

- **开源免费（MIT 协议）**：代码全公开，你不用担心它跑路。永远不会弹出"升级专业版"的提示
- **大纲式笔记（Outliner）**：每一行就是一个块。按 `Tab` 缩进，按 `Shift+Tab` 升级。不是写作文，是搭积木
- **块级引用**：不是引用整篇笔记，是引用"那一行"。想象你在新笔记里写下"正如昨天在 X 项目那行总结所说……"——点一下就到原位置
- **白板功能**：把写好的块拖到白板上，自由排布、画线连接。适合用可视化方式组织杂乱的思路
- **本地优先**：数据在你电脑上。飞机上、地铁里都能用，关掉网络照样打字

## 快速上手

### 第一步：创建第一个笔记本

打开 Logseq，选一个文件夹作为笔记本目录。Logseq 会自动在里面创建 `pages/` 和 `journals/` 两个文件夹——所有笔记都在里面，打开就能看到。

### 第二步：用大纲写第一页

新建页面，输入下面这段：

```
- AI Agent 学习计划
  - 第一周：了解工具调用
    - 什么是 Function Calling
      - 定义：模型调用外部工具的能力
  - 第二周：实践
    - 写一个天气查询的 Agent
```

按 `Tab` 缩进，按 `Enter` 创建新行。**不是写文章，是列提纲。** 你写出来的是一棵树，不是一堆字。

### 第三步：引用某一行

鼠标移到某一行，点行号前面的圆圈。复制引用链接（类似 `((block-id))`），粘贴到别的笔记里。引用的不是整篇笔记，是精确到那一行。

## 核心功能速览

**大纲编辑**：Logseq 的核心操作。Tab 缩进、Enter 换行、拖拽排序。折叠、展开，操作起来像在整理思维导图。

**块级引用与嵌入**：用 `((block-id))` 引用一个块，用嵌入语法把块嵌入到当前笔记：

```
{{embed ((block-id))}}
```

嵌入后内容实时同步——源头改了，嵌入的地方也跟着变。适合做"一句话结论多次复用"的场景。

**白板（Whiteboard）**：把写好的块拖上去。自由排布、连线、加图形和文字。适合做项目脑暴、知识地图。

**PDF 标注**：Logseq 可以直接打开 PDF，选中文字做标注。标注自动关联到你的笔记。写论文、读技术文档时特别实用。

**图谱视图**：跟 Obsidian 一样能看到笔记间的关联关系。差别是 Logseq 能看到块级别的链接——关联更细。

**查询（Query）**：用简单的查询语法搜索所有符合条件的块。比如"找出所有标签为 `#agent` 的笔记"，结果自动汇总展示。

**日记（Journal）**：默认打开就是日记页。每天的笔记自动归档到日期下。适合做每日复盘、学习日志。

## 优缺点

**优点：**
- 完全开源免费。Obsidian 的部分功能（同步、发布）要付费，Logseq 不收一分钱
- 块级引用是独门绝技。不是"引用一篇笔记"，是"引用那一行"。做知识关联时精确到行，体验完全不同
- 大纲式输入适合结构化思维。习惯后写笔记效率很高，思路不会散
- PDF 标注是内建功能。读论文、看技术文档，标注完直接跟笔记关联
- 本地优先。隐私安全，离线可用

**缺点：**
- **大纲范式有学习成本。** 习惯了"打开空白文档写文章"的人，改到"一行一行列提纲"需要时间适应。不是软件难用，是你已经习惯了另一种方式
- 界面比 Obsidian 复杂。功能多是好事情，但菜单层级有点深，新用户容易找不到按钮
- 移动端体验一般。能用，流畅度不如 Obsidian
- 插件生态不如 Obsidian 丰富。社区插件数量少，质量也参差不齐
- 数据迁移没那么简单。Logseq 虽然也存 `.md` 文件，但块 ID 是 Logseq 特有的格式，换工具时会有损失

## 一句话总结

Logseq 不让你写文章，它让你列提纲。**把你的想法拆成一块一块积木，每一块都能独立搬运、精准拼接。** 如果你习惯结构化思维，它会是你用过最趁手的知识工具。
