## Pencil 是什么

Pencil 是一款**免费、开源**的 GUI 原型设计工具,由 Evolus 团队开发维护,托管在 GitHub[^1]。它的定位是让每个人都能够轻松创建界面原型、线框图和流程图,而无需支付昂贵的商业软件许可费用。

### 适用场景

Pencil 适合以下几个典型场景:

- **线框图(Wireframing)** -- 快速勾勒应用或网页的结构布局
- **低保真原型(Low-fidelity Prototyping)** -- 在编码之前验证交互流程
- **界面设计稿(Mockup)** -- 生成可直接交付给开发者的界面示意
- **流程图和图表(Diagramming)** -- 使用内置的连接器绘制架构图和流程图

::: tip 什么时候用 Pencil?
Pencil 适合项目的**早期和中期阶段**——当你需要快速验证想法、沟通设计思路时。它不追求像素级完美,而是帮助你把注意力集中在"结构和流程"上。
:::

### 与 Figma/Sketch 的差异

| 维度 | Pencil | Figma | Sketch |
|------|--------|-------|--------|
| **价格** | 免费开源 | 免费+付费订阅 | 付费(买断+订阅) |
| **平台** | 跨平台桌面端 | Web+桌面端 | 仅 macOS |
| **协作** | 单机 | 实时多人协作 | 单机+云协作 |
| **核心定位** | 低保真原型 | 高保真设计 | 高保真设计 |
| **插件生态** | 基础(MCP扩展中) | 丰富 | 成熟 |
| **适合** | 个人开发者/小团队 | 设计团队 | 专业设计师 |

简单来说:Pencil 是"轻量级选手",它不做 Figma 和 Sketch 能做的高保真设计系统。但在**快速出原型、零成本入门**这两个指标上,它有自己的独特优势。

## 安装与界面

### 系统要求

Pencil 基于 Electron 构建(V3+版本),支持主流桌面平台:

| 平台 | 版本要求 | 下载格式 |
|------|---------|---------|
| Windows | 7 及以上 | .exe 安装包 |
| macOS | 10.9 及以上 | .dmg |
| Linux | Ubuntu 12.04+ / Fedora 21+ / Debian 8+ | .deb / .rpm / .AppImage |

### 安装方式

1. **官方下载**: 访问 [pencil.evolus.vn](https://pencil.evolus.vn/Downloads.html) 下载对应平台的安装包
2. **包管理器**: macOS 用户可通过 Homebrew 安装(`brew install --cask pencil`),Linux 用户可通过 AUR/PPA 安装
3. **源码构建**: 从 GitHub 克隆后运行 `npm install && npm start`[^2]

### 核心界面组成

启动 Pencil 后,你会看到以下主要区域:

```
┌─────────────────────────────────────────────┐
│  菜单栏 (文件/编辑/视图/工具/帮助)           │
├──────────┬──────────────────┬────────────────┤
│          │                  │                │
│  控件面板 │   画布工作区      │  属性面板     │
│  (左侧)   │   (中央)         │  (右侧)       │
│          │                  │                │
│  内置控件 │   拖拽放置        │  样式/布局    │
│  搜索过滤 │   页面切换        │  文字/背景    │
│  自定义   │   缩放控制        │  层级排列    │
│          │                  │                │
├──────────┴──────────────────┴────────────────┤
│  页面面板 (底部/左侧)                         │
│  - 页面树形结构                               │
│  - 页面增删/排序                              │
│  - 缩略图预览                                 │
└─────────────────────────────────────────────┘
```

**各区域功能**:

- **控件面板**: 包含所有可拖拽的控件元素,按类别分组,支持关键词搜索
- **画布工作区**: 核心设计区域,可拖拽控件、调整位置和大小
- **属性面板**: 选中元素后显示其样式、布局、文本等可编辑属性
- **页面面板**: 管理多页文档,支持树形结构组织页面

## 基本操作

### 创建新项目与页面

1. 启动 Pencil,点击 `文件 > 新建` 或使用快捷键 `Cmd/Ctrl + N`
2. 在弹出的模板选择对话框中,选择一个预设模板(如"空白文档"或"手机页面")
3. 在底部页面面板右键,选择"新建页面"来添加更多页面
4. 每个页面相当于一个"屏幕"或"视图",你可以通过拖拽调整页面顺序

::: tip 页面管理小技巧
使用树形页面结构组织复杂项目——例如首页下面挂子页面,让多页面原型更清晰。
:::

### 内置控件库的使用

Pencil 内置了丰富的控件集合,默认安装包括:

| 控件类别 | 包含内容 |
|---------|---------|
| **通用形状** | 矩形、圆形、三角形、线条、箭头 |
| **桌面 UI** | 按钮、输入框、下拉菜单、复选框、单选按钮、滑块 |
| **Web UI** | 浏览器窗口、导航栏、表格、分页器、搜索框 |
| **移动端** | iOS/Android 的状态栏、导航栏、按钮组、列表项、选项卡 |
| **流程图** | 开始/结束节点、决策菱形、处理框、连接线、标签 |

**操作步骤**:

1. 在左侧控件面板找到需要的控件类别
2. 点击展开类别,找到目标控件
3. 将控件**拖拽**到画布上即可完成放置
4. 释放鼠标后,控件即出现在画布上,处于选中状态

### 拖拽式设计流程

Pencil 的核心交互哲学是"所见即所得":

- **移动元素**: 选中元素后直接拖拽
- **调整大小**: 拖动元素边缘的蓝色控制点
- **旋转**: 拖动顶部的旋转手柄(圆形标记)
- **多选**: 按住 `Shift` 点击多个元素,或框选
- **对齐辅助线**: 拖拽元素时,画布会自动显示对齐参考线

### 属性编辑

选中任意元素后,右侧属性面板会显示其可编辑属性:

**样式属性**:

| 属性 | 说明 |
|------|------|
| 填充颜色 | 背景色设置,支持自定义 RGB/HEX |
| 描边颜色 | 边框颜色 |
| 描边宽度 | 边框粗细 |
| 圆角半径 | 圆角矩形控制 |
| 透明度 | 0-100% 透明度调节 |

**布局属性**:

| 属性 | 说明 |
|------|------|
| 位置(X/Y) | 元素的精确座标 |
| 尺寸(W/H) | 宽度和高度,支持锁定宽高比 |
| 层级(Z-Order) | 上层/下层/置顶/置底 |

**文本属性**(针对文字元素):

| 属性 | 说明 |
|------|------|
| 字体 | 支持系统已安装字体 |
| 字号 | 像素单位 |
| 颜色 | 文字颜色 |
| 对齐 | 左/中/右/两端对齐 |
| 加粗/斜体/下划线 | 基础文字样式 |

### 图层管理

当画布上元素较多时,图层管理至关重要:

- **右键菜单** > `排列` > `置于顶层/置于底层/上移一层/下移一层`
- 也可以通过选中元素后使用快捷键快速调整层级
- 图层顺序影响元素之间的遮挡关系——上层元素会覆盖下层元素

## 设计系统组件

### 内置模板库概览

Pencil 不仅提供基础控件,还附带了一些**完整模板**[^3]:

- 空白文档(任意尺寸)
- 手机页面(iPhone/Android 尺寸预设)
- 网页页面(常见分辨率预设)
- 对话框和弹出窗口模板

这些模板提供了预设的页面尺寸和背景,帮你省去每次手动设置尺寸的步骤。

### 创建自定义组件

当你需要在多个页面使用同一组元素时,可以创建自定义组件:

1. 在画布上设计好元素组合
2. 选中所有元素,右键 > `组合(Group)`
3. 将该组合保存为模板:选中 > `编辑 > 另存为模板`
4. 保存后,该组合会出现在控件面板的"我的模板"分类中

### 组件复用与分享

Pencil 支持通过**第三方控件集合**(Stencil Collection)扩展控件库:

- 官方下载页面提供了社区贡献的扩展控件包
- 下载后直接拖拽到 Pencil 窗口即可安装
- 安装后新控件会自动出现在控件面板中

常见的社区扩展控件包括:
- 更丰富的移动端 UI 组件
- Material Design 风格的控件库
- IoT/仪表盘专用形状
- 网络拓扑图专用图标

### 设计规范的一致性维护

通过以下策略保持设计一致性:

1. **使用同一套控件库**: 在项目启动时就确定控件来源,避免混用不同风格
2. **建立自定义模板**: 将常用的导航栏、卡片、按钮保存为模板
3. **规范颜色和字体**: 在属性面板中统一使用相同的颜色值和字体设置

## 导出与交付

Pencil 支持将设计稿导出为多种格式,满足不同交付场景的需求[^4]。

### 支持的导出格式

| 格式 | 用途 | 特点 |
|------|------|------|
| **PNG** | 图片展示、文档插图 | 栅格格式,支持透明背景 |
| **SVG** | 矢量编辑、前端使用 | 矢量格式,可缩放不失真 |
| **PDF** | 文档交付、打印 | 矢量内容,适合存档和打印 |
| **HTML** (网页) | 交互式原型 | 自动生成超链接,可点击模拟流程 |
| **ODT** | OpenOffice/LibreOffice 文档 | 直接嵌入文档 |

### 导出设置与最佳实践

**PNG 导出**:
- 推荐使用 2x 分辨率以适配 Retina 屏幕
- 单个页面导出或批量导出所有页面
- 透明背景更适合网页展示

**HTML 导出**(最推荐):
- 如果原型中设置了页面间链接,导出的 HTML 将保留这些交互
- 无需部署服务器,打开 HTML 文件即可浏览
- 适合发给客户或团队成员预览

**PDF 导出**:
- 多个页面会合并到一个 PDF 文件中
- 适合打印输出或归档存档
- 矢量内容保证打印质量

### 与开发协作交付

Pencil 导出的设计稿可以作为开发参考交付物。推荐的工作流程:

1. 使用 Pencil 完成低保真原型设计
2. 导出为 PDF 或 HTML 作为设计参考
3. 开发人员参考设计稿进行编码
4. 对于 MCP 集成场景(见下一篇文章),可通过程序化方式导出和利用设计数据

## Pencil 在 Agent 生态中的角色

### 为什么在 Agent 模块介绍设计工具?

你可能会有疑问:这是一个 Agent 生态系统模块,为什么介绍一款原型设计工具?

答案是:**
工具链的完整性**。Agent 系统不只是文字处理——它们需要和外部世界交互,包括读取、理解和生成界面设计。

Pencil 在这个生态中扮演了以下角色:

1. **MCP 集成的实验场**: Pencil 通过 MCP 协议与 AI Agent 通信,让 Agent 能够读取设计文档、操控设计元素、甚至根据自然语言指令生成界面——这在 Agent 驱动的自动化设计中至关重要。

2. **Agent 驱动设计的桥梁**: 传统的设计工具是"人→工具"的单向链路。当 Agent 通过 MCP 连接到 Pencil 后,链路变成了"人→Agent→工具"——你只需要告诉 Agent "创建一个登录页面的线框图",它就能自动操作 Pencil 完成设计。

3. **低门槛的起点**: Pencil 是开源、免费的,这意味着任何人都可以在学习 Agent 的同时,零成本地搭建一个 Agent+设计工具的集成实验环境。

### 引出 Pencil MCP

在下一篇文章中,我们将深入探讨**如何通过 MCP 协议让 AI Agent 直接操控 Pencil**——从自动生成原型到根据需求修改设计稿,你将看到 Agent 如何真正"动手"做设计。

## 本节小结

通过本节学习,你应该掌握了:

✅ **Pencil 的定位**
- 免费、开源、跨平台的 GUI 原型设计工具
- 适合线框图、低保真原型和流程图绘制
- 与 Figma/Sketch 相比,优势是零成本和学习曲线低

✅ **安装与界面**
- 支持 Windows、macOS、Linux 三大平台
- 核心界面由控件面板、画布工作区、属性面板、页面面板组成
- V3 版本基于 Electron 重构

✅ **基本操作**
- 拖拽式设计流程,所见即所得
- 丰富的内置控件库(通用形状、桌面 UI、Web UI、移动端、流程图)
- 属性编辑覆盖样式、布局、文本三大类
- 图层管理支持层级调整

✅ **设计系统组件**
- 内置模板提供快速起步
- 支持自定义组件和保存为模板
- 社区控件扩展丰富生态

✅ **导出与交付**
- 支持 PNG、SVG、PDF、HTML、ODT 格式
- HTML 导出可保留交互链接
- PDF 导出适合交付和存档

✅ **Agent 生态角色**
- 通过 MCP 协议成为 Agent 驱动设计的桥梁
- 零成本搭建 Agent+设计工具实验环境
