# Zapier 与 AI 自动化

> **学习目标**：掌握 Zapier 的注册、触发器与动作配置、AI 步骤使用、Zapier Agents 应用——最终能独立搭建一条"邮件 → AI 总结 → Notion 笔记"的完整流水线
>
> **预计时间**：40 分钟
>
> **难度等级**：⭐⭐☆☆☆

---

## 一、Zapier 是什么

先说结论：**Zapier 是自动化届的"苹果"——设计简洁、开箱即用、你不需要知道原理。**

Zapier 成立于 2011 年，到 2026 年已经有 5000+ 应用集成、300 万+ 付费用户。它的核心产品是 **Zap**（一条自动化流水线）——一个 Zap 由 Trigger（触发器）+ 一个或多个 Action（动作）组成。

2024-2026 年，Zapier 做对了三件事：
1. 加入 **AI 步骤**——原生支持 GPT-4o、Claude 等模型
2. 发布 **Zapier Agents**——AI 自主决策的工作流
3. 推出 **Zapier Central**——AI 知识库 + 自动化结合

**Zapier 最大的优势不是功能最强，而是生态最大。** 你想连什么应用，基本都有现成的 connector。

## 三、核心概念：Trigger 与 Action

### 3.1 Trigger（触发器）

触发器是 Zap 的"开工信号"。Zapier 支持四种触发方式：

| 类型 | Zappier 叫法 | 例子 |
|------|-------------|------|
| **新事件触发** | New Event | 收到新邮件、有新表单提交 |
| **定时触发** | Schedule | 每天早上 8 点、每小时一次 |
| **更新触发** | Updated Event | 数据库记录被更新时 |
| **Webhook 触发** | Catch Hook | 外部系统发 HTTP 请求时 |

配置步骤：
1. 选择触发应用（比如 Gmail）
2. 选择触发事件（比如 "New Email"）
3. 连接账号（授权 Zapier 访问）
4. 设置筛选条件（可选，比如只触发"来自特定发件人"的邮件）
5. 测试拉取数据

::: tip 第一个建议
第一次创建 Zap 时，先用 "Schedule" 触发器（定时触发）。它最简单——不需要连接其他应用，直接选"多久执行一次"。
:::

### 3.2 Action（动作）

动作是触发器之后的"干活环节"。一个 Zap 可以有多个动作（免费版最多 3 步，付费版不限）。

动作类型：

| 类型 | 描述 | 例子 |
|------|------|------|
| **创建记录** | 在应用中创建新内容 | 在 Notion 新建页面 |
| **发送消息** | 发送通知或内容 | 发送 Slack 消息 |
| **更新数据** | 修改已有数据 | 更新 Google Sheets 行 |
| **搜索/查找** | 查找已有信息 | 搜索 Notion 数据库 |
| **AI 处理** | 调用大模型处理 | 用 GPT 总结文本 |

### 3.3 Zap 的完整流程示例

```
Trigger: Gmail - New Email（收到新邮件）
    ↓
Action 1: AI - Summarize Text（AI 总结邮件内容）
    ↓
Action 2: Notion - Create Page（在 Notion 创建笔记页面）
    ↓
Action 3: Slack - Send Message（发通知："新邮件已处理"）
```

这就是 Zapier 的核心工作方式：**顺序执行，一步输出是下一步的输入。**

## 四、AI 步骤配置

Zapier 从 2024 年开始原生支持 AI 步骤。你不需要自己接 API，直接在 Zap 里选"AI"就行。

### 4.1 AI 步骤的类型

在 Create Zap 时搜索 "AI" 或 "OpenAI"，可以看到这些 AI 步骤：

| 步骤 | 干什么 | 底层模型 |
|------|--------|---------|
| **Summarize Text** | 文本总结 | GPT-4o |
| **Classify Text** | 文本分类/打标签 | GPT-4o-mini |
| **Extract Info** | 提取结构化信息 | GPT-4o |
| **Generate Text** | 根据提示生成内容 | GPT-4o |
| **Translate Text** | 翻译 | GPT-4o |
| **Improve Writing** | 润色/改写 | GPT-4o |

### 4.2 配置一个 AI 步骤

以 "Summarize Text" 为例：

1. 在 Zap 编辑器中点击 "+" 添加步骤
2. 搜索 "AI" → 选择 "AI by Zapier"
3. 选择 "Summarize Text"
4. 配置参数：

| 参数 | 说明 | 示例值 |
|------|------|--------|
| **Input Text** | 要总结的文本 | 选择上一步的邮件正文 |
| **Summary Length** | 总结长度 | Short / Medium / Long |
| **Language** | 输出语言 | Chinese |
| **Instructions** | 额外指令 | "提取关键行动项" |

5. 点击 "Test" → AI 返回总结结果
6. 结果可以被后续步骤使用（比如写入 Notion）

### 4.3 高级用法：自定义 Prompt

如果需要更灵活的控制，可以用 "Generate Text" 步骤并自定义 Prompt：

```
你是一位技术分析师。请分析以下邮件内容：

{{上一步的邮件正文}}

要求：
1. 用中文总结，不超过 200 字
2. 提取需要我回复的问题
3. 标注优先级（高/中/低）
```

::: tip 提示
Zapier 的 AI 步骤用的是 GPT-4o。不需要额外付费——AI 步骤的费用包含在 Zapier 的订阅里。但要注意免费版不支持 AI 步骤，需要付费 Plan（$19.99/月起）。
:::

## 五、Zapier Agents（自主决策）

2025 年，Zapier 发布了 **Zapier Agents**——这跟普通的 Zap 不一样了。

### 5.1 和普通 Zap 的区别

| 维度 | 普通 Zap | Zapier Agents |
|------|---------|---------------|
| **工作方式** | 你设定规则，顺序执行 | 给 AI 一个目标，AI 自主规划步骤 |
| **灵活度** | 固定流程 | AI 动态决策 |
| **适用场景** | 明确、固定、重复的任务 | 不确定、多变、需要判断的任务 |
| **学习成本** | 低 | 中等 |

大白话：**普通 Zap 是"你告诉机器每一步做什么"，Agents 是"你告诉机器想要什么结果，机器自己想办法"。**

### 5.2 使用场景举例

**场景 1：客户咨询处理**

你给 Agent 一个目标："处理今天收到的客户咨询邮件"。
Agent 自己会：
1. 读取邮件内容
2. 判断属于什么类型（投诉/询问/合作）
3. 如果是投诉 → 标记为高优先级 → 通知你
4. 如果是询问 → 从知识库找答案 → 自动回复
5. 如果是合作 → 转发到特定 Slack 频道

**场景 2：内容监测**

"每天监测 AI 相关的中文技术文章，如果发现好的就保存到 Notion 并通知我。"

### 5.3 创建 Agent

在 Zapier Dashboard 左侧菜单选择 "Agents" → "Create Agent"：

```
Agent Name: 我的 AI 助理
Goal: 每天帮我整理 Hacker News 上跟 AI 相关的文章

Tools:
├─ Web Scraper（抓取网页）
├─ AI 分析（判断文章是否和 AI 相关）
├─ Notion（保存文章）
└─ Slack（发送通知）
```

Agent 配置比 Zap 简单得多——**你不需要设计每一步的细节，只告诉它目标和可用工具就行。**

## 六、典型场景实操：邮件 → AI 总结 → Notion

这是最经典的场景，适合第一次上手。跟着做一遍。

### 6.1 准备

确保你有：
- Zapier 账号（免费版够用）
- Gmail 邮箱
- Notion 账号（没有就临时注册一个）

### 6.2 步骤

**第一步：创建新 Zap**

1. 点击 "Create Zap"
2. 命名："邮件自动总结"

**第二步：配置 Trigger**

1. 搜索 "Gmail" 作为 Trigger 应用
2. 选择事件："New Email"
3. 连接你的 Gmail 账号（授权 Zapier 访问）
4. Filter 设置：可以选择只处理特定标签的邮件（可选）
5. 点击 "Test trigger" → 确认 Zapier 能读到邮件

**第三步：添加 AI 总结步骤**

1. 点击 "+" 添加新步骤
2. 搜索 "AI" → 选择 "AI by Zapier"
3. 选择 "Summarize Text"
4. 配置：
   - Input Text：选择上一步的邮件正文
   - Summary Length：Medium
   - Language：Chinese
5. 点击 "Test" → 确认 AI 返回了总结

**第四步：保存到 Notion**

1. 点击 "+" 添加新步骤
2. 搜索 "Notion" → 选择 "Notion"
3. 选择 "Create Page"
4. 配置：
   - Database：选择你的 Notion 数据库
   - Page Title：邮件主题
   - Content：AI 总结的结果
   - 可选：添加邮件原文链接
5. 连接 Notion 账号 → 授权

**第五步：发布**

1. 点击 "Publish"
2. 确认：Zap 状态变为 "On"

### 6.3 验证

发送一封测试邮件到你的 Gmail。等待几分钟（Zapier 有 1-5 分钟的轮询间隔），检查：
- Notion 中是否自动创建了新页面
- 内容是否正确（AI 总结是否准确）
- 邮件主题是否正确显示

如果你用的是免费版，注意每月 100 次任务的限制。测试几次就花掉了额度——建议测试完先暂停 Zap，等真正要用再开启。

## 七、定价与免费额度

### 7.1 价格概览（2026 年）

| 套餐 | 月费 | 任务数 | AI 步骤 | 步数/Zap |
|------|------|--------|---------|----------|
| **Free** | $0 | 100 次/月 | 不支持 | 最多 3 步 |
| **Starter** | $19.99 | 750 次/月 | 不支持 | 最多 3 步 |
| **Professional** | $49 | 1500 次/月 | 支持 | 不限 |
| **Team** | $69/人 | 2000 次/人/月 | 支持 | 不限 |

### 7.2 花叔的省钱建议

- **学习阶段**：免费版就够了。100 次/月够你测试 5-10 条流水线。
- **个人正式用**：如果只需要普通 Zap（不用 AI 步骤），免费版或 Starter 版可行。需要 AI 步骤必须上 Professional（$49/月）—— 有点贵，但跟 n8n 的免费自部署比就是两回事了。
- **一个省钱技巧**：如果你只是少量 AI 步骤，可以自己在 Zapier 里调用 **OpenAI API 的 Webhook**，绕过 Zapier 的 AI 步骤费用。这一步有点技术门槛，但能省 $29/月。

## 八、Zapier 的优缺点

### 8.1 优点

- **上手极快**：注册到跑通第一条流水线 < 15 分钟
- **生态最大**：5000+ 应用集成，想连什么基本都有
- **稳定可靠**：SaaS 服务，不用操心运维
- **AI 集成原生**：AI 步骤不需要额外配置 API

### 8.2 缺点

- **价格贵**：AI 步骤必须 Pro 版，$49/月
- **不够灵活**：线性流程，复杂路由逻辑不好实现
- **任务限制**：免费版 100 次/月，Pro 版 1500 次/月
- **数据在云端**：敏感数据场景不适用
