# Perplexity 详解：全球最强 AI 搜索引擎

> **学习目标**：熟练使用 Perplexity 的各项功能，掌握搜索模式和追问技巧
>
> **预计时间**：30 分钟
>
> **难度**：⭐⭐

## 先说结论

Perplexity 是目前全球最强的 AI 搜索引擎，没有之一。

它的核心竞争力：**先搜再答，每句话都带出处**。不是「我猜答案是这样的」，而是「我从这 8 个网页里总结出来的，你可以点进去看原文」。

::: tip 三件套速览
- **注册**：perplexity.ai，邮箱或 Google 账号即可
- **使用**：直接在首页输入框提问，30 秒内返回答案
- **免费额度**：普通搜索无限使用；Pro Search 每天 5 次（免费版）
:::

## 注册与基础使用

### 注册流程

1. 打开 [perplexity.ai](https://www.perplexity.ai)
2. 点击右上角「Sign Up」
3. 用 Google 账号或邮箱注册
4. 完事。没有手机验证，没有复杂流程

整个过程不超过 1 分钟。

### 基础搜索

在首页的输入框直接输入你的问题。注意，是**问题**，不是关键词。

```
# 不要这样（关键词风格）
Python machine learning

# 要这样（提问风格）
我想用 Python 做一个简单的机器学习项目，预测房价，从哪里开始？
```

为什么？因为 Perplexity 的 AI 会理解你的意图，然后去搜索、整合、总结。你给的信息越多，它回答的质量越高。

::: info 关键习惯转变
从「关键词思维」切换到「提问思维」。这是用好所有 AI 搜索工具的前提。
:::

## 搜索模式详解

Perplexity 提供多种搜索模式，适用不同场景：

### 默认模式（Default）

**就是普通搜索**。适合日常快速查询。

- 搜索范围：全网
- 速度：快（5-10 秒）
- 来源数量：通常 5-8 个网页
- 适合：「什么是 RAG」「React 和 Vue 的区别」

### Focus 模式

点击输入框下方的「Focus」按钮可以切换搜索范围：

| Focus 选项 | 搜索范围 | 适合场景 |
|------------|---------|---------|
| **All** | 全网（默认） | 通用搜索 |
| **Academic** | 学术论文数据库 | 论文调研、找研究依据 |
| **Writing** | 不搜索，纯生成 | 写作辅助、创意生成 |
| **Math** | 数学计算和推导 | 解题、公式推导 |
| **Video** | YouTube 视频搜索 | 找教程、找演示 |
| **Social** | Reddit/论坛帖子 | 找真实用户评价和讨论 |

::: tip 花叔推荐
我最常用的组合：日常用 All，技术调研用 Academic，找用户真实反馈用 Social。Writing 模式其实很好用，但很多人不知道——它不搜索网页，纯粹用 AI 的知识生成内容，适合写文案、改文章。
:::

### Pro Search（原 Copilot）

Pro Search 是 Perplexity 的核心杀手锏。它不是简单搜索，而是**多步推理搜索**。

**工作流程**：

1. 你提出问题
2. Pro Search 分析你的意图，**可能反问你**以明确需求
3. 自动拆分成多个子搜索
4. 逐步搜索、整合、再搜索
5. 输出一份深度答案

**举个例子**：

```
你的问题：「帮我比较一下 2026 年主流的 AI 编程工具」

Pro Search 的执行过程：
→ 第 1 步：搜索「2026 AI coding tools comparison」
→ 第 2 步：搜索「GitHub Copilot vs Cursor vs Windsurf 2026」
→ 第 3 步：搜索「AI coding tools pricing 2026」
→ 第 4 步：整合所有信息，生成对比表格
```

整个过程大约 30-60 秒，比你自己花 2 小时翻网页强得多。

**免费版限制**：每天 5 次 Pro Search。Pro 版无限使用。

## Source 引用机制

这是 Perplexity 最重要的设计，也是它比 ChatGPT 联网更可信的原因。

**每一段回答的末尾都有数字标注**，对应右侧的来源列表：

```
AI 编程工具在 2026 年已经非常成熟 [1][2]。GitHub Copilot
的用户超过 180 万 [3]，而 Cursor 作为后起之秀，以其
深度集成 IDE 的体验获得了大量开发者青睐 [4]。

来源：
[1] GitHub Blog - State of AI in Software Development 2026
[2] Stack Overflow Developer Survey 2026
[3] GitHub 官方数据
[4] Cursor 官网博客
```

### 如何使用来源引用

1. **看标注数字**：回答中的 [1][2] 对应具体来源
2. **点击来源**：右侧面板会显示所有来源链接
3. **验证关键信息**：对重要结论，点进去看原文
4. **追问来源**：「这个数据的来源是哪里？」

::: warning 注意
Perplexity 会标注来源，但不保证 AI 总结 100% 准确。**关键信息一定要点进去验证原文**。这是 AI 搜索的基本素养。
:::

## Collections（收藏与整理）

Pro 版功能，可以创建「收藏集」来整理搜索内容。

**使用场景**：
- 做一个主题调研，把相关搜索串起来
- 收集某个领域的学习资料
- 和团队分享搜索结果

```
创建一个 Collection：
1. 点击左侧栏「Collections」
2. 新建一个集合，比如「AI Agent 调研」
3. 每次搜索后点「Save to Collection」
4. 所有搜索结果按主题归档
```

## 与 Google 搜索的使用习惯对比

| 习惯 | Google | Perplexity |
|------|--------|------------|
| 想快速了解一个概念 | 搜关键词，翻 Wikipedia | 直接问，30 秒得到总结 |
| 想找某个产品的真实评价 | 搜「XX 评价/测评」 | 用 Social Focus 搜，看 Reddit 讨论 |
| 想比较两个技术方案 | 分别搜，自己对比 | 问「比较 A 和 B」，AI 帮你做表格 |
| 想找学术论文 | Google Scholar | 切 Academic Focus，直接问 |
| 想找官网/下载链接 | Google（更直接） | Perplexity（会绕一圈） |
| 想查天气/航班 | Google（更实时） | Perplexity（也能查但不如 Google） |

**花叔的使用习惯**：日常 90% Perplexity，10% Google。Google 只用于导航型搜索（找网站、查地图、查天气）。

## 免费版 vs Pro 版

| 功能 | 免费版 | Pro 版（$20/月） |
|------|--------|------------------|
| 普通搜索 | 无限 | 无限 |
| Pro Search | 每天 5 次 | 每天 600 次 |
| 模型选择 | 默认 | GPT-4o / Claude / Sonnet 等 |
| 文件上传 | 不支持 | 支持（PDF、图片等） |
| API 访问 | 不支持 | 支持 |
| Collections | 不支持 | 支持 |
| 图片生成 | 不支持 | 支持（DALL-E 等） |

::: tip 建议
先不急着买 Pro。免费版完全够用——先用满每天 5 次 Pro Search，觉得不够再升级。大多数人日常工作用普通搜索 + 少量 Pro Search 就够了。
:::

## 实操练习

尝试用 Perplexity 完成以下搜索任务：

1. **基础搜索**：搜索「什么是 RAG 技术，用通俗的话解释」
2. **Focus 模式**：切到 Academic Focus，搜索「RAG 的最新研究进展」
3. **Pro Search**：用 Pro Search 搜索「帮我梳理 2026 年 AI Agent 领域的主要发展趋势」
4. **追问**：在第 3 个搜索结果上，追问「哪些趋势对普通开发者影响最大？」
5. **来源验证**：点开回答中的引用来源，对比原文和 AI 总结是否一致
