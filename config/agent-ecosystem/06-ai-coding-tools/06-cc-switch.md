# 工具组合与最佳实践

> **学习目标**：掌握 AI 编程工具的组合策略，"Cursor 做手、Claude Code 做脑、Codex 做腿"
> **预计时间**：50 分钟
> **难度**：⭐⭐⭐

---

## 一句话结论

不是选一个最强的工具，是让每个工具干最擅长的事。花叔的核心模型：**Cursor 做手（写代码）、Claude Code 做脑（架构思考）、Codex 做腿（安全执行）。** 三个工具加起来不到 $50/月，但效率提升 4-5 倍。不是"要不要用 AI 编程"的问题，是"怎么组合最省钱、最出活"的问题。

## 花叔的金三角模型

花叔有个著名的比喻：AI 编程工具组合就像人的三部分。

```
      ┌─────────────┐
      │ Claude Code │  ← 脑子：思考、规划、架构
      │   (终端的AI)  │
      └──────┬──────┘
             │
    ┌────────┴────────┐
    │                 │
┌───┴────┐    ┌──────┴───┐
│ Cursor  │    │  Codex   │
│ (编辑器) │    │  (沙箱)   │
│ = 手    │    │ = 腿     │
└────────┘    └──────────┘
```

### 手 — Cursor：日常编码

Cursor 负责：写代码、改代码、补全、调试、原型开发。

**为什么是手？** 手是接触世界的器官。Cursor 是你跟代码交互的主要界面。写代码、改代码、看代码——一天 80% 的时间在编辑器里。

**适合场景：**
- 日常编码
- 前端开发
- 快速原型
- 代码重构
- 阅读和理解代码

### 脑 — Claude Code：架构思考

Claude Code 负责：架构设计、大规模重构、批量操作、代码审查、自动化。

**为什么是脑？** 脑负责思考、规划、决策。Claude Code 在终端里，没有 UI 干扰，专注在"想清楚怎么做"上。

**适合场景：**
- 项目架构分析
- 大规模代码迁移
- 批量修改
- 代码审查
- 自动化流程（/loop、Hooks）

### 腿 — Codex CLI：安全执行

Codex CLI 负责：安全敏感操作、沙箱执行、第三方代码审查。

**为什么是腿？** 腿负责移动，但移动有风险——走错了得退回来。Codex 的沙箱隔离让你可以安全地"往前迈一步"，有问题随时退回。

**适合场景：**
- 安全审计
- 遗留系统改造
- 第三方代码集成
- 敏感模块修改

### 实际工作流

一个典型的工作日：

```
09:00 Cursor打开项目，看今天要做什么
09:30 Cursor写代码，遇到架构问题 → 切到 Claude Code
09:35 Claude Code分析架构，给出方案
09:40 回到 Cursor 按方案写代码
14:00 需要修改敏感模块 → 切到 Codex CLI
14:05 Codex在沙箱里执行，你审查，合并
16:00 Claude Code跑 /loop 批量修复
17:00 Cursor 收尾，提交代码
```

## 场景化组合策略

### 场景 1：一人开发（独立开发者/学生）

```
预算：$0-$20/月
推荐组合：Cursor Pro（$20）或 Trae（$0）
辅助：Claude Code Free（API 按量）
```

**花叔推荐：** 独立开发者直接上 Cursor Pro。一个月 $20 带来的效率提升至少值 $200。如果预算实在紧，Trae 免费版完全够用。

### 场景 2：小团队（3-5 人）

```
预算：$60-$200/月
推荐组合：
- Cursor Pro x 人数（$60-$100/月）
- Claude Max 5x x 1 份（$100/月）— 团队共享复杂任务
- Codex CLI（按量付费）
```

**分工：** 每人一个 Cursor 做日常开发，复杂的架构问题丢给共享的 Claude Max。

### 场景 3：企业团队（10+ 人）

```
预算：$300-$1000/月
推荐组合：
- Cursor Teams（$30/人/月）
- Claude Max 20x x 2-3 份
- Kiro（技术 Leader 级别管理 Steering Documents）
- Codex CLI（安全审查专用）
```

### 场景 4：国内开发者

```
预算：$0
推荐组合：
- Trae（免费）
- 通义灵码 / CodeBuddy（免费）
- DeepSeek API（按量，便宜）
```

## 成本计算

很多人担心多工具组合的成本。花叔算一笔账。

**假设你月薪 2 万（约 $2800），工作 22 天，每天 8 小时：**
- 时薪：约 $16/小时

**工具投入：**
| 组合 | 月费 | 相当于多少工作时间 |
|------|------|-------------------|
| 无 AI 工具 | $0 | 0 |
| Cursor Pro | $20 | 1.25 小时 |
| Cursor Pro + Claude Pro | $40 | 2.5 小时 |
| Cursor Ultra + Claude Max 5x | $300 | 18.75 小时 |
| 黄金组合（推荐） | $120 | 7.5 小时 |

**效率提升：**
- 组合 1（Cursor Pro）：效率提升 2-3 倍
- 组合 2（Pro + Claude Pro）：效率提升 3-4 倍
- 组合 3（黄金组合）：效率提升 4-5 倍

**花叔的算法：** 投入 $120/月 = 7.5 小时工资 = 赚回 120-150 小时的有效开发时间。投入产出比 1:16。

## 花叔的黄金组合方案

经过长期实践，花叔的最终推荐配置：

```yaml
主力编辑: Cursor Pro ($20/月)
  - 负责所有日常编码
  - 配置 .cursorrules 规范代码风格

大脑: Claude Code (通过 Claude Pro $20/月)
  - 负责架构分析、代码审查
  - 配置 CLAUDE.md 项目规则
  - 设置 Hooks 自动格式化
  - 创建常用 Skills

安全网: Codex CLI (API 按量 ~$5-15/月)
  - 负责安全敏感操作
  - 接收遗留系统改造任务

总成本: $45-55/月
预期效率: 3-4 倍
```

**花叔原话：** "$50 一个月换来 3 倍效率，这可能是你职业生涯性价比最高的投资。"

## CC Switch — 统一配置管理

在多种工具之间切换有个实际问题：配置分散在各处，管理麻烦。这时候需要 CC Switch。

### 什么是 CC Switch

CC Switch 是一款开源的桌面工具（GitHub 57K+ Stars），用来统一管理多款 AI 编程工具的配置。

**不是又一个 AI 编程工具，是 AI 编程工具的管家。**

### 核心功能

**1. 50+ 供应商一键切换**

内置 50+ 供应商预设，包括官方渠道（Anthropic、OpenAI）、国产模型（DeepSeek、GLM、Kimi）、中继服务（OpenRouter、SiliconFlow）。选择预设 → 填入 API Key → 点启用，5 秒完成切换。

**2. 本地 API 代理**

CC Switch 在本地启动代理服务器，接管所有 API 请求。切换模型不用重启终端，支持故障转移。

```
Claude Code → 本地代理 :15721 → 选中供应商 API
```

**3. MCP 和 Skills 管理**

统一面板管理多款应用的 MCP 服务器和 Skills，支持可视化编辑。

**4. 用量统计**

Token 消耗、缓存命中率、费用分析——帮你控制成本。

### 安装

```bash
# macOS (Homebrew)
brew tap farion1231/ccswitch
brew install --cask cc-switch

# 或从 GitHub Releases 下载
```

### 适用场景

- 你在多个 AI 编程工具之间切换
- 你使用第三方 API 中继服务
- 你需要管理多个供应商的 API Key
- 你跨多台设备开发

### 花叔评价

CC Switch 在特定场景下很有用——比如你需要频繁切换供应商、管理多个 API Key。但如果你只是用 Cursor Pro + Claude Pro 的标准组合，其实不太需要。

**花叔的法则是：不要为了管理工具的麻烦而引入另一个工具。** 先试试不加 CC Switch 的组合，觉得管理起来费劲了再上。

## 注意事项

### 避免 Tool Lock-in

不要对单一工具形成路径依赖。花叔的建议：

1. **至少掌握两个工具** — 一个主力、一个备选
2. **定期试用新工具** — 每月抽 30 分钟看看有什么新东西
3. **关注竞品动态** — Windsurf 被 Google 收购后，Cursor 加快了迭代速度。竞争对用户是好事。

### 不要过度优化

有些开发者花在配置工具上的时间比用工具编程的时间还多。

**花叔反例：** "我认识一个人，花了两周配置 MCP 服务器和 Skills，结果项目的代码一行没写。工具再强，不出活就是零。"

### 安全第一

- 不要把 API Key 提交到 git 仓库
- 对敏感代码使用 Codex CLI 的沙箱或者本地部署模型
- 定期检查 AI 生成的代码（尤其安全相关部分）

## 常见问题

**Q: 能不能只用一个工具？**
A: 可以。Cursor Pro 一个工具就能覆盖 80% 的场景。但花叔的建议是至少加一个 Claude Code，终端工具的批量操作能力 IDE 无法替代。

**Q: 组合用的成本太高？**
A: 重温上面的成本计算——$50/月换 3 倍效率，投入产出比 1:16。不是成本高，是性价比极高。

**Q: 怎么判断哪个工具适合当前任务？**
A: 一个简单的法则：
- 需要写代码 → Cursor/Trae
- 需要思考分析 → Claude Code
- 需要安全操作 → Codex CLI
- 需要管规格 → Kiro

**Q: 会不会产生工具依赖？**
A: 会。但这不是坏事。会开车的人不会因为依赖汽车而忘了怎么走路。工具依赖如果提升了你的产出，那是好事。
