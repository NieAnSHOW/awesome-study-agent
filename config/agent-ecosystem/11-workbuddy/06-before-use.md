## 一、安装与环境要求

WorkBuddy 是一款桌面端应用，免部署、下载安装即可使用。

### 系统要求

| 操作系统 | 支持版本 | 备注 |
|---------|---------|------|
| macOS | 11.0 及以上 | 提供 Intel 和 Apple Silicon（M 系列）两个版本 |
| Windows | 10 / 11 | 64 位版本 |

### 下载与安装

1. 访问官网下载页面：[https://www.codebuddy.cn/work/](https://www.codebuddy.cn/work/)
2. 根据你的操作系统选择对应版本下载
3. 安装完成后首次启动，使用腾讯云账号登录

::: tip 提示
WorkBuddy 无需任何服务器部署，下载安装后即可直接使用。安装包体积较小，下载和安装过程通常只需几分钟。
:::

## 二、积分体系

WorkBuddy 使用 Credits（积分）进行计费。了解积分体系，能帮你更好地规划使用。

### 获取积分的方式

| 途径 | 说明 |
|------|------|
| **新手福利** | 新用户注册后，前往 [https://www.codebuddy.cn/profile/usage](https://www.codebuddy.cn/profile/usage) 可领取 **5000 免费 Credits** |
| **每日签到** | 每日签到可获取额外积分，养成持续使用的习惯 |
| **社区贡献** | 在腾讯云开发者社区发布 WorkBuddy 相关文章，可获得积分奖励 |
| **邀请好友** | 邀请好友注册 WorkBuddy，邀请者和被邀请者均可获得积分奖励 |

### 注意事项

- Credits 有**有效期限制**，领取后请及时使用，避免过期浪费
- 不同模型、不同任务的 Credits 消耗量可能不同，可在 WorkBuddy 设置中查看当前积分余额和消耗明细

## 三、多模型配置

WorkBuddy 内置多模型支持，无需自备任何 API Key。

### 支持的模型

| 模型 | 特点 |
|------|------|
| **腾讯混元** | 综合表现稳定，适合日常各类任务 |
| **DeepSeek** | 代码能力强，适合编程相关任务 |
| **GLM（智谱）** | 中文理解和生成表现优秀 |
| **Kimi（月之暗面）** | 长文本处理能力强 |
| **MiniMax** | 对话和生成场景表现良好 |

### 如何切换模型

在 WorkBuddy 的设置界面中，可以一键切换当前使用的模型。不同模型在不同任务上各有优势，建议根据实际任务类型选择合适的模型：

- **编程和代码审查**：推荐 DeepSeek
- **长文档分析和总结**：推荐 Kimi
- **日常办公任务**：推荐腾讯混元或 GLM

## 四、ClawBot 远程控制

WorkBuddy 支持通过 IM（即时通讯）工具进行远程操控。这个功能叫 ClawBot，让你可以在外出时，通过手机给电脑上的 WorkBuddy 发送指令。

### 支持平台

- 企业微信
- QQ
- 飞书
- 钉钉

### 配置步骤

1. **创建 Bot**：在你使用的 IM 平台中创建一个 Bot 应用
2. **获取凭证**：记录 Bot 的 ID 和 Secret
3. **配置 WorkBuddy**：在 WorkBuddy 的设置中找到 ClawBot 配置项，填入 Bot ID 和 Secret
4. **发送指令**：配置完成后，通过 IM 向 Bot 发送自然语言指令，WorkBuddy 将接收并执行

### 典型使用场景

- 人在外面，通过手机 IM 让电脑上的 WorkBuddy 处理文件
- 远程发起任务，回到工位时结果已经准备好

::: warning 重要提醒
配置 ClawBot 时，Bot Secret 通常**只显示一次**，务必在配置时及时保存好。如果遗失，需要重新创建 Bot。
:::

## 五、定时任务

WorkBuddy 支持定时任务调度，让你的自动化工作流按计划运行。

### 支持的调度类型

| 类型 | 说明 | 示例 |
|------|------|------|
| **每日** | 每天固定时间执行 | "每天 9 点帮我生成日报" |
| **每周** | 每周固定日和时间执行 | "每周一 10 点生成上周工作周报" |
| **一次性** | 指定时间执行一次 | "下周三下午 3 点提醒我提交报销" |

### 创建方式

定时任务的创建非常直观——使用自然语言描述即可。WorkBuddy 会自动解析你的意图并创建对应的调度规则：

```
"每天 9 点帮我生成日报"
→ WorkBuddy 自动创建每日 9:00 的定时任务，关联日报生成工作流

"每月 1 号汇总上月销售数据"
→ WorkBuddy 自动创建每月 1 日的定时任务，关联数据汇总工作流
```

### 实用场景

- **每日签到提醒**：每天定时打开签到页面
- **每周工作周报**：每周五下班前自动汇总本周工作内容
- **每月数据汇总**：月初自动整理上月运营数据

## 六、安全注意事项

WorkBuddy 在设计时就考虑了安全性，提供了多层安全机制。

### 操作可追溯、可审计

所有 AI 执行的操作都有完整的日志记录，包括：
- 什么时间执行了什么操作
- 读取或修改了哪些文件
- 发送了什么网络请求

这些日志可用于事后审计和问题追溯。

### 高风险操作需人工确认

对于可能影响系统安全或数据完整性的操作，WorkBuddy 会要求人工确认后再执行。这类操作包括：

| 操作类型 | 安全策略 |
|---------|---------|
| 删除文件 | 需要人工授权确认 |
| 发送邮件/消息 | 需要人工授权确认 |
| 安装软件 | 需要人工授权确认 |
| 执行 Shell 命令 | 需要人工授权确认 |

这种设计被称为 **"人在回路"（Human-in-the-loop）**——由 AI 执行操作，但敏感步骤必须经过人的确认才能继续。

### 沙箱环境

WorkBuddy 在沙箱环境中运行，对系统核心区域有保护机制。这意味着即使出现意外情况（如提示词被恶意构造），对系统造成的潜在影响也是受限的。

### 工作目录范围限制（可选）

你可以在设置中配置 WorkBuddy 可以访问的工作目录范围，进一步缩小安全边界。建议遵循**最小权限原则**——只开放 WorkBuddy 确实需要访问的目录。

## 七、局限性说明

透明地了解 WorkBuddy 的边界，能帮你合理使用、避免踩坑。

### 复杂多步骤任务稳定性

WorkBuddy 在处理超过 20 个步骤的复杂任务时，可能会遇到超时或稳定性问题。建议将超大任务拆分为多个子任务通过定时任务依次执行。

### 专业软件兼容性

对于小众的专业软件（如 AutoCAD、Photoshop 等），WorkBuddy 的自动化兼容性有限。这些软件的自定义界面、专属快捷键和渲染方式可能导致无法正常操控。

### 技能生态仍在建设中

WorkBuddy 的技能市场还在持续丰富中。当前生态中的专门企业级技能（如直接对接 SAP、用友等企业系统的技能）还不够多。如果需要高度定制化的企业场景，可能需要自行搭建工作流。

### 积分订阅制

WorkBuddy 使用 Credits 订阅制，并非完全免费的产品。日常使用需要关注积分消耗情况，合理规划使用量。

### 输出内容需人工复核

与所有 AI 工具一样，WorkBuddy 基于大语言模型运行，存在模型"幻觉"风险——可能生成看起来正确但实际错误的内容。**对生成的结果（特别是数据汇总、财务计算、法律文书和对外发布内容）建议保持人工复核**，这是使用 AI 工具的基本原则。
